在一般人的印象中，总觉得家庭影院音响Dolby Digital（杜比数码音效）应该就是多声道,。

或者说。应该是包含主声道、中央声道、后声道与超低音声道的5.1声道环绕音效，

但事实却非如此。

录制在DVD影片中的Dolby Digital音效，存在许多可能。

它可以是最简化的单声道，

也可以是众所周知的5.1声道，

当然也有介于两者之间的Dolby Digital Stereo、Dolby Digital 4.0BDolby Digital 5.0......等方式。



在此必须补充一提的，是DVD中的声音储存方式，

除了Dolby Digital外，

也可以是DTS数码环绕音效、欧洲的MPEG 2、SONY发展的SDDS,，

或以高音质立体声为诉求的PCM数码格式等。

但目前DVD主要还是利用Dolby Digital来记录声音，营造丰富的环绕效果。

下面就是几种可能在DVD上出现的Dolby Digital格式，让我们一一解说。



Dolby Digital Mono

杜比数码单声道音效只有单声道的效果，并不足以构成立体声，**多半出现在较为古老的影片中**。

在影片制作时，**会将单声道的声音储存为AC-3格式**。

因此，当以杜比数码系统译码播放时，只有中间声道发声。

这样的音质表现,无疑是比较单调乏味的，因为连基本的立体感都十分欠缺，更遑论环绕效果。



Dolby Digital Stereo

杜比数码双声道音效将双声道的立体讯号储存为AC-3格式，

因此，当以杜比数码系统译码播放时, 可以从主声道的左右声道喇叭发声。

这其实与我们常见的两声道系统（一般音响都是如此）有相当程度的趋近，

可以建构立体的音场感,。

只不过，Dolby Digital的声音是经过压缩处理的，多少对音质有些影响。

而CD唱片则未经过压缩，拥有高音质的潜力。



Dobly Digital Surround

杜比数码环绕音效利用矩阵编码的技术，

将主声道、中央声道与后方的环绕声道挤压到两个声道中，并以AC-3格式储存。

 当以杜比数码系统译码播放时，可以聆听到完整的五声道。

不过，环绕声道为单声音效，而非立体声，而不是只有四个喇叭。

怎么会是五个声道发声呢？

那是因为，一般家庭剧院系统后方的环绕声道都是双声道，

**这里就是以双声道来呈现单声道，也就是让两个喇叭发出一模一样的声音。**

# 对象化音频

对象化音频是一种先进的音频技术，允许声音以“对象”的形式进行处理和播放，而不仅仅是传统的声道方式。以下是对象化音频的主要特点和工作原理：

### 主要特点

1. **空间定位**：
   - 声音可以在三维空间中自由放置，用户可以感知到声音的来源方向和距离。

2. **灵活性**：
   - 音频对象可以动态调整，适应不同的播放环境和设备，提供个性化的听觉体验。

3. **沉浸感**：
   - 通过在高度和宽度上增加声场，增强了听众的沉浸感，尤其是在电影院和家庭影院中。

4. **兼容性**：
   - 对象化音频技术通常与现有的音频系统兼容，能够在传统的立体声和环绕声系统上播放。

### 工作原理

1. **音频对象**：
   - 每个声音元素（如对话、音乐、效果音）被视为一个独立的对象，包含其位置、运动和音量信息。

2. **音频渲染**：
   - 播放时，系统根据用户的设备和环境，将这些音频对象渲染成适合的声道输出。

3. **动态调整**：
   - 当用户移动或调整位置时，系统实时计算并调整音频对象的位置，确保声音方向和距离变化自然。

### 应用场景

- **电影院**：提供更真实的观影体验，增强剧情的紧迫感和参与感。
- **家庭影院**：在家庭环境中实现专业级的音频体验。
- **游戏**：通过准确的声音定位增强游戏的沉浸感。
- **虚拟现实（VR）和增强现实（AR）**：提供身临其境的听觉体验。

### 代表技术

- **Dolby Atmos**：使用对象化音频技术，为用户提供三维空间的音效体验。
- **DTS:X**：类似于Dolby Atmos，支持对象化音频，增强了音效的灵活性。

# 对象化音频有没有开源的实现

是的，虽然对象化音频技术的商业实现主要由Dolby和DTS主导，但也有一些开源的实现和相关项目。以下是一些开源对象化音频的实现和资源：

### 开源实现

1. **OpenAL**：
   - **简介**：OpenAL（Open Audio Library）是一个跨平台的音频API，支持三维音频的渲染。虽然它不是专门的对象化音频实现，但可以用于创建空间音频效果。
   - **链接**：[OpenAL](https://openal.org/)

2. **Superpowered**：
   - **简介**：一个用于移动设备的音频库，支持高效的低延迟处理，适合游戏和应用程序。它允许创建空间音频效果。
   - **链接**：[Superpowered](https://superpowered.com/)

3. **Jitter**：
   - **简介**：Jitter是一个用于Max/MSP的扩展，支持空间音频和对象化音频的处理。适合音频艺术和实验。
   - **链接**：[Jitter](https://cycling74.com/products/jitter)

4. **WAVM**：
   - **简介**：一个用于Web音频的开源项目，支持对象化音频的处理，适合Web应用。
   - **链接**：[WAVM](https://github.com/WAVM/WAVM)

### 相关资源

- **Web Audio API**：
  - **简介**：虽然不是开源实现，但Web Audio API是浏览器中用于创建和处理音频的强大工具，支持三维空间音频和对象化音频模型。
  - **链接**：[Web Audio API](https://developer.mozilla.org/en-US/docs/Web/API/Web_Audio_API)

### 注意事项

- 尽管这些开源实现可以用来探索对象化音频，但它们可能不具备商业解决方案的完整功能和性能。
- 在实际应用中，可能需要结合多个库和工具来实现完整的对象化音频体验。

# 解码库和效果库是什么关系

解码库和效果库的关系可以简单理解为：**解码库是“还原声音”，效果库是“美化声音”**。它们是音频处理流程中的两个不同环节，各自功能不同，但常常协同工作。

| 类型   | 作用说明                                                | 举例                                                 |
| ------ | ------------------------------------------------------- | ---------------------------------------------------- |
| 解码库 | 将压缩的音频格式（如 AC-3、AAC）**还原成原始PCM数据**   | Dolby AC-3 解码器、DTS 解码器、FLAC 解码器           |
| 效果库 | 对PCM音频做**混音、环绕、均衡、虚拟化、增益控制**等处理 | Dolby Audio Processing、DTS:X Post Processing、EQ    |
| 关系   | 解码库输出PCM → PCM交给效果库处理 → 最终输出音频        | 解码后的音频可以选择直接播放，也可以先通过效果库优化 |

### MS12 里的关系

在 **Dolby MS12** 里，**解码库与效果库是打包在一起的**：

- **内含解码模块**：支持AC-4、E-AC-3等格式的解码。
- **内含效果模块**：支持动态范围控制、环绕声下混/上混、Dialog Enhancer（对白增强）等。

总结一句话：

> **解码库解决“听得见”，效果库解决“听得好”。**

# 除了MS12，还有什么

除了 **Dolby MS12**，还有不少类似的音频解码+处理方案，主要分为 Dolby 系、DTS 系，以及开源/其他商业方案。

| 名称                | 类型      | 说明                                                     |
| ------------------- | --------- | -------------------------------------------------------- |
| **Dolby DAP**       | 效果库    | Dolby Audio Processing，适用于移动设备和PC，侧重音效增强 |
| **Dolby Atmos**     | 解码+效果 | 支持对象音频（Object Audio），有家用/影院/移动多种版本   |
| **Dolby AC-4**      | 编码格式  | 下一代广播标准（支持场景自适应/多语言），常与MS12绑定    |
| **DTS:X**           | 解码+效果 | DTS的对象音频技术，支持自适应扬声器布局                  |
| **DTS-HD MA**       | 解码格式  | 无损音频，常用于蓝光碟                                   |
| **DTS:X Ultra**     | 效果库    | 适用于PC、游戏，偏向虚拟化环绕音效                       |
| **Open Source**     | 解码      | FFmpeg（支持AC-3、AAC、DTS等解码），无音效增强           |
| **Waves MaxxAudio** | 效果库    | 商业音效库，常用于笔电、手机                             |
| **Dirac**           | 效果库    | 高端音响优化，如房间声学补偿等                           |

### 对比思路

- **Dolby MS12**：全家桶，一体化适合家电类产品。
- **DTS:X Ultra**：效果为主，适合PC/游戏。
- **FFmpeg**：纯解码，适合嵌入式/自定义开发。
- **Dirac/Waves**：效果强，授权费高，适合高端设备。
